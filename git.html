São coisas distintas.

  • Git: Sistema de controle de versões mais popular entre os desenvolvedores.
  • GitHub: Serviço de hospedagem de repositório de software.

Git

Software para versionamento de códigos.

Documentação
Permite navegar entre versões

Útil para navegar entre as versões, localizar erros e melhorar a organização.

Possui 3 estágios fundamentais.

  • 1º estágio - Working directory: Onde trabalhamos no código.
  • 2º estágio - Staging area: Adicionamos os itens (códigos) prontos. Área de "pré-confirmação".
  • 3º estágio - Repositório: Lote pronto. Tudo ok. Oficializar a versão e guardamos.

Como usar?

Para usar o git precisamos instalar seu software disponível no link do botão.

Preciso configurar?

Sim! São duas configurações simples e globais (servirão para todos os projetos) feitas no terminal. Como usar o terminal:

  • Opção 1:
    • Windows: basta ir para Início > Executar ou Pesquisar > CMD.exe, e o nosso terminal se abrirá
    • Mac OS: ícone "Finder" situado no Dock, depois selecione "Aplicativos > Utilidades" e, finalmente, dê um duplo clique no ícone "Terminal".
  • Opção 2: Diretamente no terminal do VS Code em Menu Principal -> Terminal -> New Terminal.

Comandos

Para configurar utilize:
git config --global user.name "Seu nome aqui!"
git config --global user.email seu.email@aqui.com

Visualisar a configuração atual:
git config user.name
git config user.email

Versão do seu software

Para visualizar a versão do seu git:
git version

Tudo configurado! Como começo a usar essa ferramenta maravilhosa?

A partir daqui, os comandos são individuais para seus projetos.

git init

Vá até a pasta do seu projeto e e dentro dela utilize o git init. A partir desse momento o git sabe que tem que cuidar daquela pasta. Você já pode adicionar e remover arquivos e códigos. Trabalhe normalmente no seu projeto. Neste momento, você está trabalhando no estágio 1 - Working directory.

git status

Sempre que quiser verificar as modificações feitas no seu projeto, utilize o git status. Esse comando pode ser usado a qualquer momento. Utilize sempre que quiser se localizar. Nesta imagem por exemplo, ele informa que o index.html foi alterado.

git add .

O comando git add nome_arquivo.html envia o arquivo para o estágio 2. Quando você já terminou alguma etapa ou item da sua nova versão e já pode pré-confirmar esse arquivo. Quando a alteração envolver vários arquivos você pode usar o comando git add . e todos os arquivos pendentes serão pré confirmados. Depois que confirmar, utilize o git status novamente para ver seus arquivos confirmados.

git commit

git commit -m "Primeira versão da minha plataforma"

Depois que a versão do seu projeto estiver totalmente feita, com tudo testado e pronto para oficializar você terá uma versão pronta. É hora de guardar essa versão. O comando git add commit para o último estágio e guarda esta versão no seu repositório. Esse comando enviará para o repositório todos os arquivos que forma pré-confirmados.

Comandos interessantes

git log

Comando que permite ver os logs (histórico) do que aconteceu no repositório. Esse comando exibe todos os commits que foram feitos (por quem e quando foram feitos).

git log --oneline

Outra forma de exibir logs. Mostra os commits realizados de forma simplificada.
