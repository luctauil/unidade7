GitHub

É uma espécie de "rede social" de programadores. Com esta plataforma, podemos subir os nossos projetos e deixar que outras pessoas colaborem (ou não).

Importante: Para perfeita integração entre seu computador e o GitHub, crie sua conta com o mesmo e-mail configurado nas configurações globais do seu Git.

Crie sua conta!
Repositório

Lugar centralizado onde você pode armazenar seu código, seus arquivos e o histórico de revisão de cada arquivo. É a pasta do seu projeto.

Você consegue definir um repositório como público ou privado.

Obs: É necessário seguir as regrinhas de arquivos: não ter acento, não ter espaço, não é indicado utilizar letras maiúsculas, para separar palavras use tracinhos ou underline.

GitHub e seus repositórios na nuvem

Após criar sua conta no GitHub, para criar repositórios é só clicar em "New".

O GitHub permite guardar seus projetos na nuvem e isso traz vantagens como:

  • Possibilidade de compartilhar projetos
  • Possibilidade de trabalhar em equipe no mesmo projeto
  • Projeto salvo na nuvem gratuitamente

New Repository

Ao criar um novo repositório você precisa definir o nome do seu repositório e se ele será público ou privado.

  • Repositório público: Qualquer pessoa pode baixar e utilizar seu projeto.
  • Repositório privado: Você define quem pode visualizar, baixar e até mesmo alterar seu repositório.

Associar o projeto local com o repositório da nuvem

Depois de criar seu novo repositório você verá as informações do seu repositório vazio na nuvem. Nele tem as orientações com os comandos para que você associe a pasta do seu projeto local (projeto do seu computador) ao diretório na nuvem.

Usando branchs

Branch

Branch é uma ramificação de um repositório que permite desenvolver recursos, corrigir bugs e experimentar novas ideias de forma isolada. A palavra branch significa "ramo". Permite que vc crie ramificações no seu projeto e depois integre eles com o projeto principal.

Como trabalhar com branchs

Comandos:

  • git checkout -b nome-da-branch cria uma nova branch no seu projeto. Tudo que for feito nesta branch não impacta na main
  • git add . e git commit -m "descricao" faz commit das alterações neste novo branch.
  • git push -u origin nome-da-branch envia a branch para o repositorio remoto.
  • git checkout nome-da-outra-branch altera para qualquer outra branch
  • git merge nome-da-branch mescla a ramificação feita com a branch. Geralmente é feito na main.
  • Resolva conflitos se necessário.
